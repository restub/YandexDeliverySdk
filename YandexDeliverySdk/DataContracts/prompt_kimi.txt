Я тебе буду кидать JSON и название класса, а ты генерируй C#-классы с атрибутами WCF DataContract/DataMember/EnumMember, для использования с Newtonsoft.JSON. Не используй атрибуты JsonConverter, чтобы классы не требовали лишних зависимостей. Если целое число, используй long, если дробное — используй decimal. Не ссылайся ни на какие зависимости помимо встроенных.

Во вложении будет список уже имеющихся классов, используй их как образец. Постарайся повторно использовать уже сгенерированные классы, если у них подходящая структура и название, чтобы в итоге не было дубликатов. В API часто бывает, что одни и те же классы используются для разных ендпойнтов при обращении с одним и тем же сущностям.

Когда я перед структурой JSON пишу название раздела и ссылку на документацию, вставляй их в XML-комментарий summary перед кодом каждого класса. Смотри, как сделано в образце.

Пространство имен — YandexDeliverySdk.DataContracts. Для название C# используй PascalCase, а имена полей для DataMember бери из JSON. Nullable-типы могут быть по-разному, но чаще не nullable. Для списков пускай будут массивы, а не List<T>. Если в образце будут ISO-даты, то используй тип DateTime.
